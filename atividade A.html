1. Conceitos de POO
    1. Oque é o conceitos de POO?
      • É a programação orientada a objetos e consiste num paradigma de linguagem de programação
      • Assim como em outras areas de programação, existem diferentes caminhos para chegar ao mesmo resultado
    2. Quais linguagens tem POO?
      • Java
      • Python
      • Object Pascal
    3. Oque se aprende em POO?
      • A POO é um modelo de análise, projetos e progrmação baseado na aproximação entre o mundo real e o mundo virtual
      • Através da criação e interação entre objetos, atributos, códigos, métodos, entre outros.
  2. Quais os pilares da POO?
    1. Quem inventou a programação orientada a objetos?
      • Alan Kay
      • Autor da linguagem de programação Smalltalk
    2. Qual a função de um sistema operacionaOque são atributos no POO?
      • São as prioridades de um objeto
      • também são conecidos como variáveis ou campos
    3. Oque é contrutor em POO?
      • É um método especial que tem o mesmo nome da classe
      • E garante a alocação de mémoria necessária para cada objeto da classe
  3. Sobre as classes e objetos
    1. Oque é um objeto?
      • Os objetos possuem caracteristicas proprias
      • Os objetos pode ser categozierados, agrupados, em uma classe
    2. Qual o principal objetivo da orientação a objetos
      • Reduzir a complexidade no desenvolvimento de software
      • E aumentar sua produtividade
    3. Qual a diferença de método e atributos?
      • Atributos são as prioridades de um objeto
      • Métodos são as ações que um objeto pode realizar
  4. Sobre a linguagem POO
    1. Qual linguagem não é orientada a objetos?
      • Lisp
      • Erlang
    2. Oque seria o conceito de classe?
      • Um conjunto de objetos que compartilham as mesmas operações
      • Enquanto um objeto individual é uma entidade
    3. A importância de uma clsse em POO.
      • Ela é quem dá a vida ao objeto.
      • Nela é que são criadas as funcionalidades Através de métodos
  5. Entendendo a POO
    1. Por que as POO são necessarias?
      • POO, permitem que as pessoas compreendam prontamente o software, mesmo que não estejam familiarizadas com a implementação
      • melhora a legibilidade, a compreensão e a manutenção do código por um fator de edz
    2. As limitações da POO
      • O tamanho é maior do que o de outros aplicativos
      • A criação era trabalhosa e, em comparação com outros aplicativos, operava mais lentamente
      • Transferência de dados.
    3. Oque é polimorfismo?
      • Em POO, é um conceito fundamental
      • Ele mostra como diferentes classes podem compartilhar uma interface
      • Todas essas classes são capazes de ter implementação de interface exclusiva
